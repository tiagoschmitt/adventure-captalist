# Fix Issues

## Overview
Address feedback and fix issues in the pull request.

Read the request carefully to understand what needs to be fixed. Then:

1. Analyse the relevant code and understand the root cause
2. Make the minimal necessary changes to address the issue
3. Run lint/type/test commands if configured
4. Push a commit to the PR branch with a clear, descriptive commit message

Do not refactor unrelated code. Keep changes focused on the reported issue.

After pushing, post a brief comment summarising what was changed and why.
